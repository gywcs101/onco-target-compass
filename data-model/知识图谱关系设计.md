# 知识图谱关系设计

## 1. 核心实体

靶点、基因、蛋白、突变、通路、癌种、组织、患者亚群、生物标志物、检测方法、药物、药物形式、公司、临床试验、论文、指南、药品标签、疗效结果、安全事件、耐药事件和失败事件。

## 2. 核心关系

| 起点 | 关系 | 终点 | 必要属性 |
|---|---|---|---|
| 靶点 | 关联 | 癌种/适应症 | 证据、条件、时间 |
| 靶点 | 表达于 | 组织/细胞 | 样本、检测方法、方向 |
| 靶点 | 被药物作用 | 药物 | 结合位点、形式、机制 |
| 药物 | 开展于 | 临床试验 | 阶段、线次、状态、试验编号 |
| 试验 | 纳入 | 患者亚群 | 标志物、阈值、癌种 |
| 试验 | 报告 | 疗效/安全事件 | 终点、数值、时间、来源 |
| 论文/标签/指南 | 支持或反驳 | 结论/关系 | 原文、证据等级、适用条件 |
| 靶点/药物 | 发生 | 耐药/失败事件 | 原因、时间、证据 |

## 3. 时间与版本

每条关系都有 `valid_from`、`valid_to`、`observed_at` 和 `source_version`。新资料不覆盖旧结论，而是形成版本差异；系统显示“新事实是什么、哪条结论受影响、是否需要专家复核”。

## 4. 查询示例

“CLDN18.2 在胃癌中有哪些已验证药物形式？”查询靶点、癌种、药物形式、试验和证据关系；“KRAS G12C 的主要失败风险是什么？”查询耐药、脑转移、联合方案、安全事件和反驳关系，并按证据等级排序。
