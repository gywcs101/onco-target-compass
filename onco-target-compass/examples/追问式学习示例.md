# 追问式学习示例：从“靶点是什么”到“是否值得做”

## 1. 理解模式

**用户：** CLDN18.2 是什么？

**系统回答结构：** 先解释蛋白定位和肿瘤表达，再说明目前验证了哪类患者和药物形式，最后列出仍未解决的问题。每段事实附来源；术语提供“新人解释”和“专业展开”两个层级。

## 2. 质询模式

**系统追问：** 现有临床结果是否证明所有 CLDN18.2 表达患者都能获益？表达阈值、检测方法和治疗线次是否一致？如果疗效有限，可能是靶点、患者选择、药物暴露还是联合方案的问题？

**输出格式：** 支持证据、反向证据、证据冲突、公开证据不足、需要专家确认。

## 3. 立项模式

**用户：** 如果做 CLDN18.2 ADC，差异化从哪里开始？

**系统回答结构：** 不直接给出结论，先列出现有药物形式和竞争项目，再形成待验证假设：目标表达阈值、正常组织暴露、有效载荷、旁观者效应、给药方案和联合策略。每条假设绑定已有证据、关键缺口和建议实验。

## 4. 学习沉淀

用户阅读记录、质询答案和专家修订被保存为团队知识资产。后续用户再次提出同类问题时，系统优先展示已复核结论和版本变化。
