# 用户使用流程

## 1. 进入任务

用户输入“靶点+适应症”，也可以输入基因、突变、药物形式或自然语言问题。系统将问题识别为基础学习、证据核查、竞争分析、风险排查或立项讨论任务。

## 2. 明确分析边界

系统展示标准化后的靶点、癌种、治疗线次、患者亚群和药物形式。用户确认后进入检索；信息不足时，系统标出缺失字段，不把靶点总体结论误用于特定适应症。

## 3. 生成证据卡片

首屏给出速读结论、证据成熟度和主要风险。用户可展开机制、表达、成药性、临床、竞争和失败证据。每条结论均显示来源、发布时间、原文定位、证据类型和适用条件。

## 4. 追问式学习

理解模式用分层语言解释概念；质询模式提出“支持什么、没有证明什么、最可能在哪里失败”；立项模式把证据缺口转为可验证假设。用户可以将重要证据加入个人笔记或团队项目空间。

## 5. 输出讨论材料

系统生成证据表、竞品时间线、风险清单、差异化假设和验证问题。输出中分开显示事实、模型推断、研发建议和待专家确认项。

## 6. 更新与复核

数据源出现新试验状态、标签变化、论文或管线更新时，系统生成 Diff。用户查看变化对原结论的影响，专家确认后才更新团队版卡片。
